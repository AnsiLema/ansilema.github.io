Développer une Application Web avec Django

Dans ce projet, j’ai développé une application web permettant aux utilisateurs de publier et demander des critiques de livres et d’articles. J’ai utilisé le framework Django pour implémenter le rendu côté serveur, gérer l’authentification des utilisateurs et structurer la base de données conformément au cahier des charges.

Février 2025

Développer une API sécurisée avec Django REST Framework

Dans ce projet, j’ai créé une API RESTful performante et sécurisée en utilisant le framework Django REST. J’ai défini les modèles de données en suivant des diagrammes d’analyse, conçu une architecture robuste et structuré proprement les différentes applications du projet.

J’ai mis en place un système d’authentification sécurisé avec OAuth2 et JWT, géré finement les permissions d’accès aux ressources, et respecté les normes de protection des données RGPD et de sécurité OWASP. Grâce à l'utilisation d'outils tels que Postman, j’ai validé l’intégralité des fonctionnalités de l’API via des tests précis des endpoints.

Par ailleurs, j’ai optimisé l’application en appliquant les principes du Green Code : j’ai intégré un système de pagination pour limiter le poids des réponses API et minimiser la consommation de ressources. Mon projet est entièrement versionné sur GitHub avec un fichier README clair et complet, et utilise Pipenv pour une gestion fiable des dépendances.

Enfin, cette expérience m’a permis d’approfondir mes compétences en modélisation, en sécurité des API, en optimisation de performances, et en respect des bonnes pratiques de développement durable.

Dans ce projet, j’ai amélioré une application web existante développée avec le microframework Flask, en suivant une approche rigoureuse basée sur la méthode Test-Driven Development (TDD). J’ai mis en place une suite complète de tests unitaires et d’intégration avec pytest, atteignant une couverture de code de 100%.

J’ai corrigé les principaux bogues critiques de l’application en améliorant la gestion des erreurs et des exceptions pour garantir une expérience utilisateur fluide et sécurisée. Chaque correctif a été réalisé sur une branche dédiée, en suivant une nomenclature Git professionnelle, avec un processus de validation rigoureux avant fusion.

J’ai également optimisé l’interface utilisateur afin de prévenir toute action non conforme (limitation des réservations à 12 places, gestion des points disponibles, accès sécurisé aux compétitions). Le projet est entièrement versionné sur GitHub avec une organisation soignée et une documentation claire.

Cette expérience m’a permis de renforcer mes compétences en debugging Python, en qualité logicielle, en automatisation des tests et en gestion professionnelle des projets avec Git et GitHub.
